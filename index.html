Step-by-Step Installation

1

Extract the Files

After downloading the ZIP file, right-click it and select "Extract All...". Place the extracted folder somewhere safe (like your Documents or Desktop).

2

Enable Developer Mode

Open Chrome and type chrome://extensions in your URL bar. In the top right corner, toggle Developer mode to ON.

3

Load Unpacked

Click the Load unpacked button in the top left. Select the extracted folder from Step 1. The extension will now appear in your list!

4

Authenticate & Add API Key

Log in to your IARE Lake Dashboard to automatically unlock the extension. Then, click the extension icon, paste your free Gemini API Key, and click Save. Refresh your Edpuzzle tab (F5) to apply changes!

How to Get Your Gemini API Key

To let the Auto-Pilot solve your Edpuzzle quizzes, you need a free API key from Google AI Studio. Follow these steps to generate yours securely.

1

Open Google AI Studio

Go to the official Google AI Studio website and sign in using your Google account.

2

Create a New Project & Key

Navigate to the API keys section on the left sidebar and click on "Create API key". Select "Create API key in new project" to isolate your token safely.

3

Save It in the Extension

Copy your unique alphanumeric key, open the IARE Lake extension popup on your browser toolbar, paste it into the Gemini API Key field, and click Save.

🛡️ Critical Safety Rules: Never Share Your API Key!
Your API key acts like a password to your developer account. If someone else gets it, they can abuse your quota.

What to do:
* Never paste your raw API key directly into public code repositories, GitHub commits, or public chat forums.
* Rest assured, the IARE Lake extension stores your key securely inside your browser's local storage (chrome.storage.local). It is never sent to external servers or third-party databases.

Understanding the Auto-Pilot Logs

Curious how the AI works behind the scenes? By opening Chrome Developer Tools (F12), you can watch the system dynamically solve questions and manage API rate limits in real-time.

1. The AI Fallback System (Handling Rate Limits)

Because the free Gemini API has strict Request Per Minute (RPM) limits, fast playback speeds can trigger a Quota Exceeded error. Our system is designed to catch this and instantly switch to a backup model so your video never stops.

What this image shows: On the left, the primary gemini-3.6-flash model hit a quota limit (yellow warning). The system instantly bypassed the error and successfully used the backup model.

2. The Auto-Solve Lifecycle

When a question appears, the system extracts the text, sends it to the AI, verifies the selection, clicks the target option, submits the answer, and clicks continue.

What this image shows: The right-side console confirms the full lifecycle. You will see markers for [AI Auto-Pilot] ➜ Selecting... followed by ✅ Selection verified and ✅ Submitting.

Troubleshooting Guide

If the extension stops working, check the exact symptoms below to fix the issue instantly.

🚨 Symptom: Stuck on "Please log in to your IARE Lake Dashboard"
Reason for Failure: Chrome content scripts do not inject into tabs that were already open before the extension was installed.

What to do: Go to your open IARE Lake tab and hard refresh the page by pressing F5.

How to check: Press F12 on the dashboard to open DevTools, go to the Console, and look for this exact success message:
[IARE Lake Bridge] Auth active. Roll: [Your Roll Number]
⚠️ Symptom: Extension fails to save Settings / Status turns RED
Reason for Failure: The extension validates your API key before saving. If the key is incorrect, the save is blocked.

What to do: Ensure there are no spaces at the start or end of your API key.

How to check: Open the DevTools Network tab on the Edpuzzle page, click Save in the popup, and look for the models?key= network request. If it returns 400 Bad Request, the key is invalid.
🚨 Symptom: Video Speed won't change / Extension seems dead
Reason for Failure: Any time you update, reload, or reinstall the extension in chrome://extensions, previously open tabs lose connection to the background worker.

What to do: You must refresh the Edpuzzle tab (F5) to re-attach the new script.

How to check: Open the Console on the Edpuzzle page. If you see Uncaught TypeError: Cannot read properties of undefined (reading 'sendMessage'), your tab is orphaned.
✅ Symptom: AI Logs show yellow Quota Exceeded warnings
Reason for occurrence: The free Gemini API limits rapid requests. When questions appear back-to-back at 16x speed, the primary model can get temporarily blocked.

What to do: Do nothing! This is intended behavior. The system automatically shifts to fallback models.

How to check: As seen in the verification images above, look below the yellow warning for a log reading [AI Auto-Pilot] ➜ Trying: gemini-3.5-flash-lite.
